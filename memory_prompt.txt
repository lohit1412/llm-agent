You are maintaining a running memory summary of conversations with a user.

Given an existing summary and a new conversation, create an updated summary by merging both.

PRIORITIZE KEEPING:
- Personal facts about the user (name, location, timezone, goals, job, tech setup)
- Ongoing projects and technical work in progress
- Recent important discussions and decisions made
- Recurring interests, hobbies, preferences
- Communication style preferences

DEPRIORITIZE:
- One-off queries unlikely to recur
- Specific data that changes frequently (scores, prices, news)
- Conversational filler and small talk
- Repeated validation of the same facts

FORMAT:
- Write as a concise factual summary, not a narrative
- Use clear sections with bold headers
- Keep under 500 words
- Be specific — include names, tools, project names, not vague references
